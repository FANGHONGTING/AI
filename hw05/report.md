# 实验五：CNN手写数字识别实验报告

## 一、文章学习摘要

学习了微信公众号文章《计算机视觉》第10篇：极简卷积神经网络CNN识别手写数字。

### 1. CNN核心组件

- **卷积层**：通过卷积核提取图像局部特征（如边缘、纹理）
- **激活函数**：引入非线性，增强网络表达能力（使用ReLU）
- **池化层**：降采样减少特征图尺寸，降低计算量（使用最大池化）
- **全连接层**：将提取的特征映射到分类结果

### 2. SimpleCNN模型结构

| 层 | 操作 | 输入 | 输出 |
|----|------|------|------|
| 输入层 | - | 28×28×1 | 28×28×1 |
| 卷积层 | 3×3, 16个, padding=1 | 28×28×1 | 28×28×16 |
| ReLU激活 | - | 28×28×16 | 28×28×16 |
| 池化层 | 2×2, 步长2 | 28×28×16 | 14×14×16 |
| 全连接层 | - | 3136 | 10 |

### 3. 训练流程

1. 数据加载：从torchvision下载MNIST，归一化处理（均值0.1307，标准差0.3081）
2. 前向传播：图像 → 卷积 → ReLU → 池化 → 展平 → 全连接 → 输出
3. 损失计算：交叉熵损失函数
4. 反向传播：计算梯度
5. 参数更新：Adam优化器，学习率0.001
6. 训练5个epoch，批量大小64

### 4. 模型参数

- 总参数量：31,530
- 卷积层：160个参数
- 全连接层：31,370个参数

## 二、LeNet-5结构说明

LeNet-5是由Yann LeCun提出的经典卷积神经网络，最初用于手写数字识别。针对MNIST数据集（28×28），实现结构如下：

| 层 | 操作 | 输入尺寸 | 输出尺寸 | 参数量 |
|----|------|----------|----------|--------|
| 输入 | - | 28×28×1 | 28×28×1 | 0 |
| C1 | 卷积5×5, 6个, padding=2 | 28×28×1 | 28×28×6 | 156 |
| S2 | 最大池化2×2, 步长2 | 28×28×6 | 14×14×6 | 0 |
| C3 | 卷积5×5, 16个 | 14×14×6 | 10×10×16 | 2,416 |
| S4 | 最大池化2×2, 步长2 | 10×10×16 | 5×5×16 | 0 |
| C5 | 卷积5×5, 120个 | 5×5×16 | 1×1×120 | 48,120 |
| F6 | 全连接84 | 120 | 84 | 10,164 |
| 输出 | 全连接10 | 84 | 10 | 850 |
| **总计** | - | - | - | **61,706** |

### 设计思路

- 采用两个卷积-池化块提取层次化特征
- 卷积核大小5×5，padding=2保持C1层输出尺寸不变
- C5层使用卷积代替全连接，减少参数量
- 最后通过两个全连接层映射到10个类别

## 四、极简CNN与LeNet-5对比

| 对比项 | 极简CNN | LeNet-5 |
|--------|---------|---------|
| 测试准确率 | 98.04% | 98.92% |
| 参数量 | 31,530 | 61,706 |
| 训练耗时（5轮） | 约2分18秒 | 约2分55秒 |
| 卷积层数 | 1 | 2 |
| 池化层数 | 1 | 2 |
| 全连接层数 | 1 | 3 |

### 对比分析

1. **准确率**：LeNet-5（98.92%）比极简CNN（98.04%）高约0.9%。原因是LeNet-5有更深的网络结构（2个卷积层+2个池化层），能提取更丰富的层次化特征。

2. **参数量**：LeNet-5（61,706）约为极简CNN（31,530）的2倍。主要增加在全连接层（C5、F6、输出层），模型表达能力更强，但需要更多计算资源。

3. **训练耗时**：LeNet-5（约2分55秒）比极简CNN（约2分18秒）多约37秒，与参数量增加成正比。

### 结论

LeNet-5通过增加网络深度和宽度，在参数量翻倍的情况下换取了约0.9%的准确率提升。对于MNIST这种相对简单的任务，极简CNN性价比更高；而LeNet-5的结构为更复杂的图像识别任务奠定了基础。

## 五、调试记录

### 错误1：pip安装网络超时

- **现象**：执行 `pip install` 时报网络连接错误
- **原因分析**：默认PyPI源在国外，网络不稳定
- **修改点**：使用清华镜像源成功安装

### 错误2：ModuleNotFoundError: No module named 'torch'

- **现象**：运行 `python simple_cnn.py` 提示找不到torch
- **原因分析**：电脑有多个Python环境，torch只安装在其中一个
- **修改点**：使用完整路径 `C:\Anaconda\python.exe` 运行

### 总结
两个模型均成功运行，环境配置问题已解决。
