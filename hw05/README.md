# 实验五：CNN手写数字识别

## 任务一：极简CNN

### 文章要点概括

这篇文章讲解了如何使用PyTorch构建一个极简的卷积神经网络来识别MNIST手写数字。

**核心知识点：**
1. **CNN的四个核心组件**：卷积层（提取特征）、激活函数（引入非线性）、池化层（降采样减少计算量）、全连接层（分类）

2. **SimpleCNN模型结构**：
   - 输入：28×28灰度图像
   - 卷积层：1通道→16通道，3×3卷积核，padding=1（保持尺寸不变）
   - ReLU激活
   - 最大池化层：2×2窗口，步长2（28×28→14×14）
   - 全连接层：3136维 → 10维（0-9分类）

3. **训练流程**：
   - 数据加载：从torchvision下载MNIST，归一化处理
   - 前向传播：图像→卷积→ReLU→池化→展平→全连接→输出
   - 损失计算：交叉熵损失函数
   - 反向传播：计算梯度
   - 参数更新：Adam优化器，学习率0.001
   - 训练5个epoch，批量大小64

4. **模型参数**：总共31,530个可训练参数

## 任务二：LeNet-5

### 网络结构

LeNet-5是由Yann LeCun提出的经典卷积神经网络，最初用于手写数字识别。针对MNIST数据集（28×28），我实现了以下结构：

| 层 | 操作 | 输入尺寸 | 输出尺寸 | 参数量 |
|----|------|----------|----------|--------|
| 输入 | - | 28×28×1 | 28×28×1 | 0 |
| C1 | 卷积5×5, 6个, padding=2 | 28×28×1 | 28×28×6 | 156 |
| S2 | 最大池化2×2, 步长2 | 28×28×6 | 14×14×6 | 0 |
| C3 | 卷积5×5, 16个 | 14×14×6 | 10×10×16 | 2,416 |
| S4 | 最大池化2×2, 步长2 | 10×10×16 | 5×5×16 | 0 |
| C5 | 卷积5×5, 120个 | 5×5×16 | 1×1×120 | 48,120 |
| F6 | 全连接84 | 120 | 84 | 10,164 |
| 输出 | 全连接10 | 84 | 10 | 850 |
| **总计** | - | - | - | **61,706** |

### 训练配置

| 配置项 | 值 |

| 训练轮数 | 5 |
| 批量大小 | 64 |
| 优化器 | Adam |
| 学习率 | 0.001 |
| 使用设备 | CPU |

## 任务三：模型对比

| 对比项 | 极简CNN (SimpleCNN) | LeNet-5 |
|--------|---------------------|---------|
| 测试准确率 | 98.04% | 98.92% |
| 参数量 | 31,530 | 61,706 |
| 卷积层数 | 1 | 2 |
| 池化层数 | 1 | 2 |
| 全连接层数 | 1 | 3 |
| 5轮训练时间 | 约2-3分钟 | 约3-4分钟 |

### 分析

1. **准确率对比**：LeNet-5的准确率（98.92%）高于极简CNN（98.04%），提升了约0.9%。这是因为LeNet-5拥有更深的网络结构（2个卷积层+2个池化层），能够提取更丰富的层次化特征，从而获得更好的分类效果。

2. **参数量对比**：LeNet-5的参数量（61,706）约为极简CNN（31,530）的2倍。LeNet-5的全连接层（C5、F6、输出层）贡献了大部分参数增量，这使得模型具有更强的表达能力，但也需要更多的计算资源和训练时间。

3. **结论**：LeNet-5通过增加网络深度和宽度，在参数量翻倍的情况下换取了约0.9%的准确率提升。对于MNIST这种相对简单的任务，极简CNN已经能达到98%以上的准确率，性价比更高；而LeNet-5的结构设计为更复杂的图像识别任务奠定了基础。
