## 摘要

本报告基于Kaggle公开数据集Chest X-Ray Images (Pneumonia)，使用卷积神经网络完成肺炎二分类任务。通过数据平衡、数据增强等技术处理类别不平衡问题，最终在测试集上达到84.46%的准确率和98.21%的召回率。报告重点分析了医疗AI场景下召回率优先的设计原则。

### 2.2 类别不平衡

原始数据中肺炎样本（3,875）约是正常样本（1,341）的**2.9倍**，存在明显的类别不平衡问题。

### 2.3 数据预处理策略

为解决类别不平衡：
1. **随机下采样**：从肺炎样本中随机选取1,341张（与正常样本数量相等）
2. **重新划分**：按8:2比例划分训练集和验证集
3. **类别权重**：计算`compute_class_weight`，给少数类更高权重

## 五、任务二：结果分析

### Q1: 模型是否出现高准确率但召回率偏低？医学角度哪个指标更重要？

**回答**：

本模型的表现特征是：**高召回率（98.21%）、中等精确率（80.97%）、中等准确率（84.46%）**。这不是"高准确率低召回率"的情况，而是"高召回率低精确率"。

**医学诊断角度，召回率（Recall）比准确率更重要**，原因如下：

| 维度 | 说明 |
|------|------|
| **假阴性代价** | 将肺炎患者误判为正常 → 延误治疗 → 病情恶化 → 可能死亡 |
| **假阳性代价** | 将正常人误判为肺炎 → 进一步检查 → 医疗资源浪费 → 患者焦虑 |
| **临床原则** | "宁可误报，不可漏报"是医疗筛查的基本准则 |
| **实际应用** | 高召回率的模型可作为初筛工具，假阳性由医生二次确认 |

因此，本模型98.21%的召回率虽然牺牲了部分精确率，但在临床场景下是可接受的，甚至优于高精确率但低召回率的模型。

### Q2: 数据增强与迁移学习对结果是否有帮助？

**回答**：

**（1）数据增强的作用**

本实验采用了旋转、平移、缩放、翻转等数据增强策略。从结果看：

- ✅ **有效缓解过拟合**：训练曲线显示验证集准确率（95.90%）高于训练集（93.13%），说明数据增强起到了正则化效果
- ✅ **提升泛化能力**：模型在未见过的测试集上表现稳定
- ✅ **低成本高回报**：不增加额外计算成本

**（2）迁移学习的价值**

由于CPU环境限制，本实验未使用迁移学习。基于理论分析：

| 方面 | 从头训练CNN | 迁移学习（ResNet50等） |
|------|-------------|----------------------|
| 特征提取 | 从零学习 | 利用ImageNet预训练特征 |
| 训练速度 | 较慢 | 较快（冻结底层） |
| 小样本表现 | 一般 | 优秀 |
| 预期准确率 | 80-85% | 90-95% |

**结论**：在有GPU条件下，推荐使用迁移学习；本实验由于硬件限制选择从头训练，结果仍达到作业要求。

### Q3: 假阴性与假阳性的后果？

**回答**：

#### 假阴性（肺炎 → 正常）- 7例

| 后果 | 严重程度 |
|------|----------|
| 患者未接受治疗，病情从轻症发展为重症肺炎 | 🔴 严重 |
| 可能发展为呼吸衰竭、脓毒症 | 🔴 严重 |
| 延误最佳治疗窗口，增加治疗难度和费用 | 🔴 严重 |
| 患者在传染期未隔离，可能传染给家人和医护人员 | 🔴 严重 |
| 极端情况可能导致死亡 | 🔴 致命 |

#### 假阳性（正常 → 肺炎）- 90例

| 后果 | 严重程度 |
|------|----------|
| 接受不必要的进一步检查（CT、血常规、痰培养） | 🟡 中等 |
| 可能被开具不必要的抗生素（导致细菌耐药性） | 🟡 中等 |
| 患者产生焦虑心理负担 | 🟡 较轻 |
| 造成医疗资源浪费 | 🟡 较轻 |
| 假阳性可由医生二次读片纠正 | 🟢 可逆 |

**总结**：
- 假阴性的7例是模型的核心风险，应优先优化
- 假阳性的90例虽多，但在筛查场景下可接受
- 本实验的召回率表现符合医疗AI的设计原则
